1. OVERVIEW & SYSTEM ARCHITECTURE
================================================================================
AlphaCapital is a real-time portfolio management and paper trading application.
It allows users to monitor real-time stock price fluctuations, maintain a custom
watchlist, simulate stock transactions (BUY/SELL) with virtual paper cash, review
auto-generated AI stock recommendations, and inspect portfolio risk analysis reports.

High-Level Architecture:
------------------------
                  +--------------------------------+
                  |         React Frontend         |
                  |     (Vite Dev Server, 5173)    |
                  +---------------+----------------+
                                  |
            HTTP/REST Requests    |   Real-Time Price Updates
           (JWT Bearer Token header) |   (WebSockets: ws://...)
                                  v
                  +---------------+----------------+
                  |        Express Backend         |
                  |         (NodeJS, 5000)         |
                  +---------------+-------+--------+
                                  |       |
      PostgreSQL Queries          |       |   Redis get/setEx
      (or In-Memory Mock DB fallback)     |   (or In-Memory Mock Cache fallback)
                                  v       v
                  +---------------+--+ +--+---------------+
                  | PostgreSQL DB    | | Redis Cache (60s) |
                  | (Postgres Port)  | | (Memurai, 6379)   |
                  +------------------+ +-------------------+

Dual-Mode Reliability (Fail-safe Mechanisms):
----------------------------------------------
* Database Fail-safe: If PostgreSQL connection fails on startup, the application
  automatically switches to an in-memory Mock Database storage, initializing the
  same schema logic seamlessly.
* Caching Fail-safe: If the Redis server is offline, the backend falls back to
  an in-memory Map object acting as a TTL-aware Cache.


2. DATABASE STRUCTURE (PostgreSQL & Mock Fallback)
================================================================================
File Reference: [db.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/db.js)

The system manages 5 tables, automatically generated on system startup:

1. `users` Table:
   - Tracks user credentials and virtual cash balances.
   - Fields:
     * `id`: SERIAL PRIMARY KEY
     * `email`: VARCHAR(255) UNIQUE NOT NULL (User identity)
     * `password`: VARCHAR(255) NOT NULL (Bcrypt-hashed password)
     * `balance`: DECIMAL(15, 2) DEFAULT 100000.00 (Available paper trade cash)
     * `created_at`: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

2. `portfolios` Table:
   - Tracks active stock holdings owned by each user.
   - Fields:
     * `id`: SERIAL PRIMARY KEY
     * `user_id`: INTEGER REFERENCES users(id) ON DELETE CASCADE
     * `ticker`: VARCHAR(10) NOT NULL (e.g., 'AAPL', 'NVDA')
     * `shares`: DECIMAL(12, 4) DEFAULT 0.0000 (Quantity of shares owned)
     * `average_buy_price`: DECIMAL(12, 2) DEFAULT 0.00 (Weighted average cost)
     * CONSTRAINT: UNIQUE(user_id, ticker) (One entry per stock per user)

3. `transactions` Table:
   - Historical ledger of executed trade orders.
   - Fields:
     * `id`: SERIAL PRIMARY KEY
     * `user_id`: INTEGER REFERENCES users(id) ON DELETE CASCADE
     * `ticker`: VARCHAR(10) NOT NULL
     * `type`: VARCHAR(10) NOT NULL ('BUY' or 'SELL')
     * `shares`: DECIMAL(12, 4) NOT NULL
     * `price`: DECIMAL(12, 2) NOT NULL
     * `timestamp`: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

4. `watchlists` Table:
   - Custom stocks pinned by users for rapid dashboard reference.
   - Fields:
     * `id`: SERIAL PRIMARY KEY
     * `user_id`: INTEGER REFERENCES users(id) ON DELETE CASCADE
     * `ticker`: VARCHAR(10) NOT NULL
     * CONSTRAINT: UNIQUE(user_id, ticker)

5. `ai_recommendations` Table:
   - Cache of current AI buy/hold/sell signals and quantitative scores.
   - Fields:
     * `id`: SERIAL PRIMARY KEY
     * `ticker`: VARCHAR(10) UNIQUE NOT NULL
     * `recommendation`: VARCHAR(10) NOT NULL ('BUY', 'SELL', 'HOLD')
     * `score`: INTEGER NOT NULL (0-100 rating scale)
     * `rationale`: TEXT NOT NULL (Contextual market analysis)
     * `updated_at`: TIMESTAMP DEFAULT CURRENT_TIMESTAMP


3. BACKEND ROUTERS & REST API ENDPOINTS
================================================================================
File Reference: [server.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/server.js)

All REST routes are protected using the `authenticateToken` JWT middleware which
intercepts requests, extracts the authorization header ("Bearer <token>"), decodes
the user profile payload, and appends `req.user` to subsequent handlers.

Here is the breakdown of routers mounted on `/api`:

A. Auth Router (`/api/auth`)
   File Reference: [routes/auth.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/routes/auth.js)
   * `POST /api/auth/register`:
     - Creates a new user record in the DB with default $100,000 cash balance.
     - Hashes the password using Bcrypt.
     - Connectivity: Returns JSON profile on success. Switches frontend modal
       from registration to login mode.
   * `POST /api/auth/login`:
     - Validates credentials against DB. Generates a signed JWT access token.
     - Connectivity: Frontend stores this token in LocalStorage and AppContext.
   * `GET /api/auth/me`:
     - Fetches active user metadata (ID, email, and current cash balance).
     - Connectivity: Used on app reload to maintain session persistence.

B. Portfolio Router (`/api/portfolio`)
   File Reference: [routes/portfolio.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/routes/portfolio.js)
   * `GET /api/portfolio/overview`:
     - Combines user's holdings with their current prices to compute:
       * Cash balance, holdings value, net portfolio value.
       * Total cost basis, total P&L, and total return percentages.
     - Connectivity: Feeds the Portfolio and Dashboard frontend views.

C. Stocks & Watchlist Router (`/api/stocks`)
   File Reference: [routes/stocks.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/routes/stocks.js)
   * `GET /api/stocks`:
     - Returns names, tickers, drift parameters, and volatilities of all stocks.
   * `GET /api/stocks/:ticker/price`:
     - Retrieves the active price of a specific stock ticker.
   * `GET /api/stocks/:ticker/history`:
     - Generates and returns 30 days of historical daily price candles (date,
       open, high, low, close, volume) for high-performance chart mapping.
   * `GET /api/stocks/watchlist/list`:
     - Retrieves all stock symbols watched by the authenticated user.
   * `POST /api/stocks/watchlist/add`:
     - Inserts a target ticker symbol into the user's watchlist array.
   * `DELETE /api/stocks/watchlist/remove/:ticker`:
     - Deletes a stock symbol from the user's watchlist database rows.

D. Trading Simulator Router (`/api/trading`)
   File Reference: [routes/trading.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/routes/trading.js)
   * `POST /api/trading/order`:
     - Processes order requests: `ticker`, `type` (BUY/SELL), and `shares`.
     - Validates cash sufficiency (for BUY orders) or shares ownership (for SELL).
     - Modifies user balance. Updates holdings via weighted average cost updates
       if buying, or decrementing shares if selling.
     - Connectivity: Logs transaction in `transactions` table. Updates user cache.
   * `GET /api/trading/transactions`:
     - Returns a list of past user trades sorted chronologically.

E. AI Analytics Router (`/api/ai`)
   File Reference: [routes/ai.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/routes/ai.js)
   * `GET /api/ai/recommendations`:
     - Fetches AI recommendations from DB. Seeds default ratings on first load.
   * `POST /api/ai/recalculate`:
     - Re-evaluates ratings using minor random drift offsets to simulate active
       market analysis and updates DB rows.

F. Risk Reports Router (`/api/reports`)
   File Reference: [routes/reports.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/routes/reports.js)
   * `GET /api/reports/generate`:
     - Computes risk analytics metrics:
       * Beta estimation (weighted average of stock betas: AAPL 1.2, MSFT 0.9,
         TSLA 2.1, NVDA 1.8, AMZN 1.15, GOOGL 1.05, BTC 3.0, CASH 0.0).
       * Sharpe ratio estimate and portfolio volatility risk rating (Low/Med/High).
       * Complete diversification weight mapping (including CASH allocation).
       * Tailored optimization advice depending on allocation thresholds.

G. Admin Users Router (`/api/admin`) [Created Router File]
   File Reference: [routes/admin.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/routes/admin.js)
   * `GET /api/admin/users`:
     - Fetches details of all users (ID, email, balance, created_at).


4. WEBSOCKET REAL-TIME STREAM & SIMULATION LOOP
================================================================================
File References: [services/market.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/services/market.js),
[websocket/socket.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/websocket/socket.js)

AlphaCapital features a live simulation engine simulating stock pricing cycles
every 2 seconds:

Market Fluctuation Service (`market.js`):
-----------------------------------------
1. Baseline Setup: Seeds default values for: AAPL, MSFT, TSLA, NVDA, AMZN, GOOGL, BTC.
2. Background Loop: Runs `setInterval(fluctuatePrices, 2000)`.
3. Geometric Brownian Motion (GBM) Formula:
   - For each stock: `pctChange = drift + volatility * random_normal`
   - Updates active session prices in-memory and pushes the updates to Redis
     with a 60-second expiration.
4. Broadcast Subscription: Subscribes WebSocket connections to the updater.

WebSocket Protocol (`socket.js`):
---------------------------------
1. Standard HTTP Upgrade: Listens for `/ws` handshake requests, upgrading
   incoming TCP sockets.
2. Initial Handshake: On a new client connection, the server queries the system
   for all current prices and sends a `'INITIAL_PRICES'` payload immediately.
3. Broadcast Dispatch: Every 2 seconds when prices change, the market service callback
   fires. The server packages this as a `'PRICE_UPDATES'` type message and
   broadcasts it to all active, open socket connections.
4. Client Handlers: If connection breaks, client reconnects automatically every 5s.


5. REDIS CACHING MECHANISM
================================================================================
File Reference: [cache.js](file:///c:/Users/sukun/Desktop/Stock%20Market/backend/cache.js)

To prevent DB strain and ensure rapid query response, Redis is used as an
intermediate caching layer:

Caching Rules:
--------------
* Key Structure: `stock:<ticker>:price` (holds active stock price).
* Set Expiration: Caches values for 60 seconds using `client.setEx(key, 60, price)`.
* Fetch Pipeline:
  - When `getPrice(ticker)` is called:
    1. Checks if Redis holds the key. If yes, returns cached price.
    2. If no, reads current memory price, saves it to Redis, and returns it.


6. FRONTEND SINGLE PAGE APPLICATION (SPA) STRUCTURE
================================================================================
Folder Reference: [frontend/src](file:///c:/Users/sukun/Desktop/Stock%20Market/frontend/src)

The React client coordinates components, style contexts, and server routes:

A. Global State Manager (`AppContext.jsx`)
   - Initializes connections and acts as global context Provider.
   - State Variables:
     * `user`: Logged-in profile credentials and cash balance.
     * `token`: Active JWT authorization token.
     * `activeTab`: Target dashboard view.
     * `prices`: Object mapping stock tickers to real-time prices.
     * `stocks`: List of available securities.
     * `watchlist`: Monitored stock symbols.
     * `portfolio`: Holdings and performance statistics.
     * `transactions`: Trade ledger.
     * `aiRecs`: Current AI analysis scores.
   - Interceptors: If an API returns a 401/403 status (Expired session), the context
     automatically calls `logout()`, removing local variables to secure the workspace.
   - Sync: Pulls portfolio performance data every 5 seconds to sync values.

B. Interface Components
   * `LiveTickerTape.jsx`: Scrolling ticker marquee at the top showing real-time price updates.
   * `AuthModal.jsx`: Login & Register panels with custom password input controls.
   * `DashboardView.jsx`: High-level metrics, gainers, losers, and watchlist overview cards.
   * `PortfolioView.jsx`: Aggregated holdings table, asset balance card, and transaction log.
   * `MarketDataView.jsx`: Stock browser list. Selecting a stock draws a interactive canvas chart
     (`StockChart.jsx`) and shows the order desk trade module.
   * `WatchlistView.jsx`: View watched stock quotes and manage watchlist cards.
   * `AiRecommendationsView.jsx`: Signal ratings (Buy/Sell/Hold) with recalculate control.
   * `RiskReportsView.jsx`: Visual allocation ratios, portfolio beta, Sharpe calculations,
     and auto-generated risk mitigation instructions.
   * `TradingSimulatorView.jsx`: Trade order ticket directly submitting buy/sell orders.
   * `ToastNotification.jsx`: User message alerts.


7. STEP-BY-STEP DATA FLOWS & USER INTERACTION CYCLES
================================================================================

A. Authentication Cycle
-----------------------
[User submits Register form]
   --> Sends POST `/api/auth/register`
   --> DB validates uniqueness, hashes password, saves record
   --> Client receives success response, switches to Login view
[User submits Login form]
   --> Sends POST `/api/auth/login`
   --> Backend validates email/password, issues signed JWT token
   --> Client stores token in LocalStorage and AppContext.token state
   --> Context triggers `loadUserData()`: fetches user profile, portfolio, watchlist, and AI recommendations
   --> View switches to Dashboard

B. Real-Time Price Update Cycle
------------------------------
[Backend timer fluctuates prices (2s)]
   --> Fluctuates prices using Drift and Volatility values
   --> Saves new prices to Redis cache (60s expiry)
   --> Calls WebSocket broadcast handlers
   --> Sends payload to connected client WebSockets
   --> Client `AppContext` receives update in `onmessage` callback
   --> Updates `prices` state
   --> React triggers re-render of components showing real-time prices (Ticker, Market, Portfolio)

C. Paper Trading Order Cycle
----------------------------
[User submits BUY order for 10 NVDA shares in Simulator]
   --> Client invokes `executeTrade('NVDA', 'BUY', 10)` in context
   --> Sends POST `/api/trading/order` (JWT in headers)
   --> Backend queries user cash balance and current NVDA price from Redis cache
   --> Validates cash balance >= 10 * current NVDA price
   --> Deducts total cost from user's DB balance
   --> Recalculates user's portfolio holding average cost and shares count
   --> Saves trade log to `transactions` table
   --> Returns success response
   --> Client receives response, re-fetches Portfolio and Transactions, triggers toast alert

D. Watchlist Toggle Cycle
-------------------------
[User clicks Star icon next to AAPL]
   --> Client context checks if AAPL is in watchlist state
   --> If NOT: Sends POST `/api/stocks/watchlist/add`
   --> Backend writes user ID and 'AAPL' into watchlists DB table
   --> Context appends 'AAPL' to state array; star icon changes color
   --> If YES: Sends DELETE `/api/stocks/watchlist/remove/AAPL`
   --> Backend deletes user's AAPL row from watchlists DB table
   --> Context filters out 'AAPL' from state array
